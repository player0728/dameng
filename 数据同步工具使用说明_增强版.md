# 数据同步工具使用说明（增强版）

## 一、概述

`data_sync_tool_plus.py` 是一个统一的数据同步配置向导工具，集成了数据同步和 Git 仓库管理功能（更新为dameng_test.py）（更新为dameng_new.py）（更新为git_new.py）（更新为data_new.py）。


### 支持的功能模块

| 序号 | 功能模块 | 说明 |
|-----|---------|------|
| 1 | 数据同步 | SeaTunnel 配置向导，支持多种数据源 |
| 2 | Git 仓库同步 | 克隆、检查更新、拉取代码 |

### 数据同步支持的数据源类型

| 序号 | 数据源类型 | 目标类型 | 说明 |
|-----|-----------|---------|------|
| 1 | MySQL 数据库 | MySQL 数据库 | 支持 CDC 实时同步和批量同步，自动获取库/表/字段结构，支持字段映射，自动创建目标表 |
| 2 | HTTP API | MySQL 数据库 | 支持随机用户 API 和自定义 API，自动解析 JSON 结构 |
| 3 | 达梦数据库 | 达梦数据库 | 达梦数据库之间的数据同步，自动获取库/表/字段结构，支持字段映射，自动创建目标表 |
| 4 | 达梦 CDC 实时监控 | 达梦数据库 | 基于触发器的达梦到达梦实时同步，支持配置多个监控任务，支持查看状态、停止、删除 |
| 5 | 测试模式 | Console | 使用 FakeSource 生成测试数据 |

## 四、数据同步功能详解

### 4.1 MySQL 数据库同步

#### 支持的作业模式
- **STREAMING**：实时同步（CDC 模式）
- **BATCH**：批量同步

#### 配置步骤
1. 输入作业名称
2. 选择作业模式
3. 输入并行度
4. 配置源 MySQL 数据库连接信息
5. 配置字段映射（可修改目标字段名、类型，删除或添加字段）
6. 配置目标 MySQL 数据库连接信息
7. 确认执行

#### 自动功能
- 自动检测可用数据库列表
- 自动检测数据库中的表列表
- 自动获取表结构并生成字段映射
- 自动创建不存在的目标表
- **同步历史记录**：自动记住上次同步配置，下次同步可直接使用

#### 同步历史记录功能
当执行 MySQL → MySQL 同步时，工具会自动保存字段映射配置到 `config/sync_history.json` 文件。下次同步相同的源表和目标表时，会提示：
```
发现同步历史记录！
上次同步配置：seatunnel_prod.testtable->target.test_1
是否使用上次的字段映射配置？[y/N]: 
```
如果选择"是"，则自动使用上次的字段映射，无需重新配置。

### 4.2 HTTP API 同步

#### 支持的 API 类型
- **Random User Generator API**：预置的随机用户数据 API（https://randomuser.me/）
- **自定义 API**：用户自定义的 API 接口

#### 自定义 API 配置
- API URL：API 接口地址
- HTTP 方法：GET 或 POST
- JSON 内容字段：数据所在的 JSON 字段路径（支持嵌套，如 `data.items`）

#### 自动功能
- 自动测试 API 连接
- 自动解析 API 返回的 JSON 结构
- 自动提取字段名和类型

### 4.3 达梦数据库同步

#### 支持的功能
- ✅ 自动获取数据库列表
- ✅ 自动获取表列表
- ✅ 自动获取表字段信息（Schema）
- ✅ 字段映射配置（可修改目标字段名、删除不需要的字段）
- ✅ 自动创建不存在的目标表（根据字段映射配置）

#### 配置步骤
1. 配置源达梦数据库连接信息（主机、端口、用户名、密码）
2. 自动获取并显示可用的数据库列表，选择数据库
3. 自动获取并显示该数据库中的表列表，选择表
4. 自动获取并显示所选表的字段结构（Schema）
5. 字段映射配置（可以修改字段名、删除不需要的字段）
6. 配置目标达梦数据库连接信息
   - 如果输入的表不存在，会询问是否根据字段映射自动创建
7. 确认执行同步

#### 自动创建目标表
如果目标表不存在，会自动创建：

```

#### 同步逻辑
- **数据相同**：跳过（不执行任何操作）
- **主键相同，数据不同**：更新
- **主键不存在**：插入

| INT | INT |
| DATETIME | DATETIME |
| BOOLEAN | BOOLEAN |
| STRING | VARCHAR(255) |

## 五、达梦 CDC 实时监控功能详解

### 5.1 功能概述

达梦 CDC 实时监控功能基于数据库触发器实现，能够实时捕获源表的增删改操作，并同步到目标表。

#### 支持的特性

- ✅ 基于触发器的实时数据捕获
- ✅ 支持配置多个监控任务
- ✅ 自动检测表主键
- ✅ 支持查看监控任务状态
- ✅ 支持删除监控任务
- ✅ 支持停止监控任务
- ✅ 每次启动自动清理旧变更日志表
- ✅ 任务列表显示当前状态（运行中/已停止）

### 5.4 启动 CDC 监控服务

启动时会自动：
1. 删除旧变更日志表
2. 创建新变更日志表
3. 删除旧触发器（如果存在）
4. 创建新触发器（INSERT、UPDATE、DELETE）
5. 开始实时监控

启动示例：
```
【启动 CDC 监控服务】

已配置的 CDC 监控任务：

  1) TEST.USERS → TEST.TARGET [stopped]

请选择要启动的任务 [1-1]: 1

创建变更日志表...
ℹ 已删除旧的变更日志表
✓ 变更日志表创建成功

为表 'USERS' 创建 CDC 触发器...
✓ 表 'USERS' 的 CDC 触发器创建成功

✓ CDC 监控已启动，正在监控变更...

按 Ctrl+C 停止监控

检测到 3 个变更
  ✓ UPDATE: PK=1
  ✓ UPDATE: PK=2
  ✓ UPDATE: PK=3
```

### 5.5 查看监控状态

查看已配置的监控任务及其当前状态。

### 5.6 停止监控服务

停止正在运行的监控服务。

### 5.7 删除监控任务

删除不需要的监控任务配置。

### 5.8 工作原理

1. **触发器捕获**：在源表上创建 3 个触发器（INSERT、UPDATE、DELETE），捕获数据变更
2. **变更日志**：触发器将变更记录保存到变更日志表（DAMENG_CDC_CHANGE_LOG）
3. **轮询同步**：监控循环定期查询变更日志表，将未处理的变更同步到目标表
4. **标记处理**：同步完成后，标记变更记录为已处理

### 5.9 数据同步逻辑

| 操作类型 | 处理逻辑 |
|---------|---------|
| INSERT | 在目标表插入相同数据 |
| UPDATE | 根据主键更新目标表数据 |
| DELETE | 根据主键删除目标表数据 |

## 六、Git 仓库同步功能详解

### 6.1 克隆远程仓库

```

  3) 停止 Git 监控服务

  7) 返回主菜单

```

### 6.2 配置 Git 监控任务

配置步骤：
1. 输入仓库地址（HTTPS/SSH）
2. 输入本地路径
3. 输入分支名
4. 选择监控模式：
   - **自动拉取**：检测到更新后自动拉取
   - **手动拉取**：检测到更新后通知，由用户决定是否拉取
5. 设置轮询间隔（秒）

#### 监控模式说明

| 模式 | 说明 |
|------|------|
| 自动拉取 | 检测到更新后自动拉取，无需人工干预 |
| 手动拉取 | 检测到更新后通知用户，用户输入"更新"才拉取 |

## 八、注意事项

1. **环境变量**：建议设置 `SEATUNNEL_HOME` 环境变量指向 SeaTunnel 安装目录
2. **达梦 ODBC 驱动**：使用达梦数据库同步功能需要提前安装 DM8 ODBC 驱动
3. **Git 安装**：使用 Git 功能需要提前安装 Git 客户端
4. **网络访问**：克隆远程仓库需要网络访问权限
5. **数据安全**：请妥善保管数据库密码和 Git 仓库凭证
